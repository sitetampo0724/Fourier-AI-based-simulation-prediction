RMSE: 0.004354 —— 均方根误差
含义：预测值与真实值的典型偏差幅度
计算公式：
plain
RMSE = √(平均(预测值 - 真实值)²)
特点：
对大误差惩罚更重（平方放大）
单位与输出相同（谐波幅度的单位）
直观理解：
典型情况下，预测幅度和理论幅度相差约 0.004。比如理论值是 0.500，预测值通常在 0.496~0.504 之间。
MAE: 0.001175 —— 平均绝对误差
含义：预测值与真实值的平均绝对偏差
计算公式：
plain
MAE = 平均(|预测值 - 真实值|)
特点：
直接反映平均偏差，更直观
不受极端值影响
直观理解：
平均来说，预测值只比真实值偏了 0.001。这个误差非常小。

负值	打乱后性能反而更好 → 特征含噪声，正常时干扰了模型

好问题！这暴露了你这个ML模型的一个本质局限。
先看数据
表格
案例	幅度	1次谐波相对误差	整体误差
1	1.0	0.52%	~0.4%
2	2.0	2.41%	~2.3%
3	1.5	0.99%	~1.0%
不是简单的"幅度越大误差越大"，而是 2.0 这个点刚好落在模型学得不好的区域。
根本原因：树模型不擅长做"乘法"
你的模型是 Gradient Boosting（梯度提升树），它的决策方式是：
plain
if amplitude < 1.2? → 走左边
if amplitude > 1.8? → 走右边
树模型天生擅长"分类判断"，不擅长"连续乘法"。
理想公式是：
plain
幅度 = 4A/(nπ)  →  A 和结果是严格的乘法关系
但树只能分段近似这个乘法：
plain
A=0.5 → 输出 0.63  (记住这个值)
A=1.0 → 输出 1.27  (记住这个值)  
A=1.5 → 输出 1.90  (插值: 还行)
A=2.0 → 输出 2.49  (插值: 偏了2.4%)
A=3.0 → 输出 3.78  (可能又偏了)
2.0 刚好卡在几棵树的分裂边界上，每棵树给一个小误差，800棵树累积下来就偏了 ~2.4%。
三个具体原因
表格
原因	解释
1. 树模型做乘法有固有误差	决策树是分段常数函数，无法精确表达 k×A 的线性关系
2. 训练样本太少	只有200个样本，幅度覆盖不均匀，2.0附近可能没学到精确的映射
3. 特征中幅度"太直接"	input_amplitude 直接进模型，但GBDT把它当"分类条件"用，而不是乘数
验证：误差模式是"比例缩放"
仔细看案例2：
表格
阶数	理论值	预测值	偏低比例
1次	2.546	2.485	-2.4%
5次	0.509	0.498	-2.3%
15次	0.170	0.165	-2.6%
31次	0.082	0.080	-2.1%
所有阶数同比例偏低约2.3%！这说明模型对幅度A的预测整体打了一个折扣——它学到了幅度大致是多少，但没精确到乘法比例。
